# Story 1.5: Parser SPPREV Pensionistas

**Status:** InReview
**Story ID:** 1.5
**Epic:** 1 - Core Engine
**Priority:** High
**Complexity:** High

## User Story

Como usuario,
eu quero que o sistema extraia corretamente todos os dados de holerites SPPREV de pensionistas,
para que eu possa processar holerites de pensionistas do SPPREV.

## Acceptance Criteria

1. Parser SPPREV Pensionista implementado seguindo a interface base_parser
2. Extrai cabecalho especifico: nome, CPF, cargo ex-servidor, entidade, tipo folha, data pagamento, benefício, competencia, cota parte, banco/agencia/conta
3. Estrutura UNICA com 2 seções: BASE DE CALCULO DO BENEFÍCIO + DEMONSTRATIVO DO PAGAMENTO DO BENEFÍCIO
4. BASE DE CALCULO: código (XXXXXX), denominação, vencimentos, descontos
5. DEMONSTRATIVO: código (XXXXXX), denominação, período, vencimentos, descontos (markers -C = crédito, -D = débito)
6. Extrai totais: total vencimentos (base calc), total descontos (base calc), líquido (base calc), total vencimentos (demonst), total descontos (demonst), liquido (demonst)
7. Template detector identifica pela presenca de "PENSÃO" + "DEMONSTRATIVO DE PAGAMENTO"
8. Testes com holerites SPPREV pensionistas reais

## Dev Notes

- Reutilizar `BaseParser` da Story 1.3 - não reimplementar
- SPPREV Pensionista: estrutura DIFERENTE de SPPREV Aposentado (2 seções obrigatórias)
- Usar `pdf_reader.PaginaExtraida` como input, retornar `data_model.Holerite`
- Regex patterns devem ser case-insensitive e tolerantes a espaçamento
- Valores monetários em formato brasileiro: 1.234,56 (ponto = milhares, vírgula = centavos)
- Códigos de verba: formato XXXXXX sem ponto (similar a Aposentado)
- Marcadores -C (crédito) e -D (débito) para diferenciar natureza na seção DEMONSTRATIVO
