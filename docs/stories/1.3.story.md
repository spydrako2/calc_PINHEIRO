# Story 1.3: Parser DDPE (Template Específico)

**Status:** Draft
**Story ID:** 1.3
**Epic:** 1 - Core Engine
**Priority:** High
**Complexity:** High

## User Story

Como usuário,
eu quero que o sistema extraia dados específicos de holerites DDPE (Departamento de Despesa) de forma confiável e estruturada,
para que eu possa processar verbas, descontos e totais com precisão.

## Acceptance Criteria

1. Módulo `parsers/ddpe_parser.py` implementado com classe `DDPEParser` que herda de `BaseParser`
2. Extração de cabeçalho: CPF, Nome, Cargo, Unidade, Competência, Tipo de Folha, Data de Pagamento
3. Extração de verbas: código, denominação, natureza (NORMAL/ATRASADO/REPOSIÇÃO), quantidade, valor
4. Extração de totalizações: Total Vencimentos, Total Descontos, Líquido
5. Detecção automática de layout DDPE por template matching (score >= 0.75)
6. Suporte a múltiplas páginas com detecção de continuação automática
7. Testes com 3+ holerites DDPE reais (cobertura >= 80%)
8. Validação de dados extraídos (CPF formato, valores numéricos, etc)

## Dev Notes

- BaseParser é interface que outros parsers (SPPREV_APOSENTADO, SPPREV_PENSIONISTA) implementarão
- DDPE é template mais comum nos dados de referência - começar por este
- Usar `pdf_reader.PaginaExtraida` como input, retornar `data_model.Holerite`
- Regex patterns devem ser case-insensitive e tolerantes a espaçamento
- Valores monetários em formato brasileiro: 1.234,56 (ponto = milhares, vírgula = centavos)
- Códigos de verba: usar `normalizer.CodigoVerbaNormalizer` para padronização

## Testing

- Usar PDFs reais de `docs/referencias/` (DDPE holerites)
- Validar que todos os campos obrigatórios são extraídos
- Testar com holerites de múltiplas páginas
- Testar edge cases: verbas com valores zero, descontos faltantes, etc
